Week 7 - IoT, ADC/DAC, Arduino IDE & ESP-IDF

Week 7 - IoT, ADC/DAC, Arduino IDE & ESP-IDF

Overview

This week, we expanded our knowledge of embedded systems by diving into the Internet of Things (IoT), analog-to-digital conversion, and professional development tools for microcontroller programming.

Internet of Things (IoT)

We were introduced to the fundamentals of IoT — the network of physical devices embedded with sensors, software, and connectivity that enables them to collect and exchange data. We learned how embedded systems form the backbone of IoT applications, connecting the physical world to the digital realm.

Analog-to-Digital Converter (ADC) & DAC

We explored the ADC, which converts analog signals from sensors into digital values that microcontrollers can process. We also learned about Digital-to-Analog Converter (DAC), which does the reverse — converting digital values back into analog signals for output devices like speakers or motors.

Understanding ADC and DAC is essential for interfacing with real-world sensors and actuators that operate with continuous analog signals.

Arduino IDE

We learned to use the Arduino IDE for writing, compiling, and uploading code to microcontroller boards. The Arduino IDE provides a user-friendly environment with a vast library ecosystem, making it accessible for rapid prototyping and embedded development.

ESP-IDF (ESP32 IoT Development Framework)

We were introduced to ESP-IDF, the official development framework for ESP32 microcontrollers. ESP-IDF is a powerful tool that allows developers to write firmware in C/C++ with full access to the hardware features of the ESP32, including Wi-Fi, Bluetooth, and advanced peripheral control.

Hands-on: Blinking the Inbuilt LED

Using C programming, we were able to flash code to the ESP32 and blink the inbuilt LED. This practical exercise helped us understand the basics of GPIO (General Purpose Input/Output) control and how to write low-level embedded code.

Debugging

We were also shown how to debug our code using various techniques and tools. Debugging is a critical skill for identifying and fixing issues in embedded firmware, ensuring reliable operation of microcontroller applications.

Pins Basics

We learned about the different types of pins on microcontrollers — GPIO pins, power pins, analog pins, and communication pins (UART, SPI, I2C). Understanding pin mapping and configuration is essential for correctly connecting external components.

Key Learning Outcomes

  • Understanding the fundamentals of the Internet of Things (IoT)
  • Learning how Analog-to-Digital Converters (ADC) and Digital-to-Analog Converters (DAC) work
  • Using the Arduino IDE for embedded development
  • Working with ESP-IDF for ESP32 development
  • Flashing C code to blink the inbuilt LED
  • Learning debugging techniques for embedded systems
  • Understanding microcontroller pin types and configurations

Overall, Week 7 was an exciting continuation of our embedded systems journey, equipping us with the skills needed to build IoT applications and work with professional-grade development tools.
