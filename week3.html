Week 3 - Microcontroller Programming & Simulations

Overview

In Week 3, we dove into microcontroller programming and gained hands-on experience through simulations using TinkerCAD and other simulation platforms.

Microcontroller Programming

We learned how to write code for microcontrollers, focusing on the Raspberry Pi Pico and similar platforms. This included understanding how to control GPIO pins, read sensor inputs, and drive output devices based on programmed logic.

TinkerCAD Simulations

We used TinkerCAD, a web-based 3D design and simulation tool, to create virtual microcontroller circuits and test our code without needing physical hardware.

Through TinkerCAD, we were able to:

  • Design circuit layouts with microcontrollers and components
  • Write and upload code to the virtual Pico
  • Observe the output based on the data and code provided
  • Test different scenarios and edge cases
  • Debug and refine our programs in a safe environment

Other Simulation Platforms

We also explored other simulation tools that allowed us to model microcontroller behavior and visualize how the code interacts with virtual hardware components.

Pros and Cons of Simulations

Advantages

  • No need for physical hardware to get started
  • Safe environment to experiment and make mistakes
  • Quick iteration and testing of code
  • Cost-effective for learning and prototyping
  • Easy to share and collaborate on projects

Disadvantages

  • Does not fully replicate real-world hardware behavior
  • Timing and real-time constraints may differ from actual hardware
  • Limited accuracy in simulating analog signals and sensor noise
  • Cannot test physical properties like heat, vibration, or power consumption
  • Some peripheral devices may not be available in simulation

Week 3 gave us valuable experience in prototyping and testing microcontroller projects virtually, while also teaching us the limitations of simulation-only environments.
