Week 2 - C Programming & Assembly Language

Overview

In Week 2, we shifted focus from high-level web technologies to the foundational programming languages used in embedded systems and microcontroller development.

C Programming Language

We learned the C programming language, which is the industry standard for embedded systems development. C provides low-level hardware access while maintaining portability across different microcontroller platforms.

We covered the basics of C including:

  • Variables and data types
  • Control structures (if/else, loops)
  • Functions and modular programming
  • Pointers and memory management
  • Bitwise operations

Assembly Language

We were also introduced to assembly language, the lowest-level programming language that directly corresponds to machine code instructions. Assembly language gives us complete control over the microcontroller's hardware.

Key assembly concepts included:

  • Registers and memory addressing
  • Instruction sets
  • Assembly syntax and mnemonics
  • Interfacing with hardware peripherals

Programming for Controllers

We learned how to write programs specifically for microcontrollers and embedded systems. This included understanding hardware constraints, optimizing code for limited resources, and working with peripherals like GPIO pins, timers, and interrupts.

Basics of Rules in Embedded Systems

We explored the fundamental rules and best practices for embedded systems programming:

  • Memory efficiency and optimization
  • Real-time constraints and timing
  • Hardware abstraction layers
  • Debugging and testing embedded code

Week 2 provided us with the essential programming skills needed to develop firmware for embedded devices, bridging the gap between software and hardware.
