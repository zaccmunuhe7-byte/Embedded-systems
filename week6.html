Week 6 - IoT, ADC & DAC

Overview

Week 6 marked a pivotal moment in our embedded systems journey — we transitioned from hardware-focused topics to the conceptual frameworks that enable connectivity and analog-digital signal conversion. This week endowed us with understanding of the Internet of Things paradigm and the signal processing techniques essential for interfacing with the physical world.

Internet of Things (IoT)

The Internet of Things (IoT) constitutes the technological framework whereby physical objects — sensors, actuators, appliances, and vehicles — are embedded with computing capability and network connectivity, enabling them to collect, exchange, and act upon data without human intervention.

What is IoT?

At its core, IoT describes a network of interconnected devices that communicate via the Internet, sharing telemetry data and receiving command instructions. These devices typically comprise:

  • Sensors: Devices that convert physical phenomena (temperature, pressure, motion, light) into electrical signals
  • Microcontrollers: Processing units that read sensor data, perform computations, and control actuators
  • Communication Modules: Wi-Fi, Bluetooth, LoRa, or cellular radios that enable network connectivity
  • Cloud Platforms: Remote servers that aggregate data, perform analytics, and provide management interfaces

How IoT Works

  1. Data Acquisition: Sensors continuously measure physical quantities and convert them to electrical signals
  2. Signal Conditioning: Analog signals are amplified, filtered, and converted to digital format
  3. Local Processing: The microcontroller processes data, applies algorithms, and makes immediate decisions
  4. Transmission: Processed data is transmitted to cloud platforms via Wi-Fi, cellular, or LPWAN networks
  5. Cloud Analytics: Remote servers store, analyze, and visualize data
  6. Action: Commands are sent back to actuators to effect physical changes

Advantages of IoT

  • Remote Monitoring: Observe system status from anywhere in the world
  • Automation: Eliminate manual intervention in routine processes
  • Predictive Maintenance: Detect anomalies before failures occur
  • Resource Optimization: Minimize energy consumption and material waste
  • Scalability: Deploy thousands of devices from a central management platform
  • Data-Driven Decisions: Base operational choices on quantitative evidence rather than intuition

Challenges and Considerations

  • Security: Networked devices present attack surfaces for malicious actors
  • Privacy: Sensitive data collection raises user privacy concerns
  • Interoperability: Diverse device manufacturers use incompatible protocols
  • Power Consumption: Continuous connectivity demands significant energy
  • Data Management: Massive telemetry volumes require efficient storage and processing
  • Reliability: Network outages can disable critical systems

Analog-to-Digital Converter (ADC)

The Analog-to-Digital Converter is a peripheral that transforms continuous analog voltage signals into discrete digital values that microcontrollers can process, store, and transmit.

What is an ADC?

Real-world phenomena — temperature, pressure, light intensity, sound — manifest as continuous analog signals. Microcontrollers, however, operate on binary data. The ADC bridges this divide by sampling analog signals at discrete intervals and representing each sample as a numerical value.

How ADC Works

  1. Sampling: The ADC measures the input voltage at regular intervals (sampling rate)
  2. Quantization: The measured voltage is compared to reference voltage levels and assigned a digital code
  3. Encoding: The digital code is converted to binary format for the processor
  4. Output: The digital value is stored in a register for software access

Key ADC Parameters

  • Resolution: The number of bits in the output — a 12-bit ADC provides 4096 possible values (212)
  • Sampling Rate: The number of conversions per second (samples per second)
  • Reference Voltage: The maximum voltage that maps to the maximum digital output
  • Input Channels: The number of analog inputs that can be sampled

ADC on the Pico W

The Raspberry Pi Pico W features a 12-bit ADC with four input channels — GPIO 26, 27, and 28 are dedicated analog inputs, while channel 3 is connected to the internal temperature sensor.

Advantages of ADC

  • Enables microcontrollers to interact with analog sensors
  • Provides precise numerical representation of physical quantities
  • Facilitates signal processing and data analysis
  • Supports closed-loop control systems

Limitations of ADC

  • Quantization error introduces measurement uncertainty
  • Aliasing occurs if sampling rate is insufficient for signal frequency
  • Noise can degrade measurement accuracy
  • Limited resolution may be insufficient for precision applications

Digital-to-Analog Converter (DAC)

The Digital-to-Analog Converter performs the inverse operation of the ADC — transforming digital values into corresponding analog voltage or current signals.

What is a DAC?

While ADCs convert sensor readings into numbers, DACs convert numbers into physical actions — driving speakers, controlling motor speed, adjusting brightness, or generating test signals.

How DAC Works

  1. Input: A digital value (from 0 to 2n-1 for n-bit DAC) is loaded into the DAC
  2. Conversion: The DAC generates an output voltage proportional to the input value
  3. Scaling: The output is scaled such that maximum input produces the reference voltage
  4. Smoothing: For audio applications, reconstruction filters smooth the stepped output

Applications of DAC

  • Audio playback (speakers, headphones)
  • Motor speed control
  • LED brightness control (PWM as pseudo-DAC)
  • Signal generation for testing
  • Voltage-controlled oscillators
  • Programmable power supplies

Advantages of DAC

  • Enables digital systems to control analog actuators
  • Provides variable output for continuous control
  • Facilitates signal generation and synthesis
  • Supports feedback control systems

Limitations of DAC

  • Output is quantized — not truly continuous
  • Limited resolution affects precision of control
  • Output bandwidth may be insufficient for high-frequency applications
  • Some DACs require external amplification

Key Learning Outcomes

  • Comprehension of IoT architecture, advantages, and challenges
  • Understanding ADC operation, parameters, and applications
  • Understanding DAC operation, parameters, and applications
  • Knowledge of signal conversion between analog and digital domains
  • Foundation for building connected sensor systems
  • Understanding of quantization, sampling, and aliasing concepts

Week 6 provided the conceptual foundation for understanding how embedded systems interact with both the physical world and the digital infrastructure of the Internet — essential knowledge for any IoT practitioner.
